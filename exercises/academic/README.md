# Academic Scenarios · 學術寫作場景

四個圍繞「論文/學術寫作」的子場景。每組從中**擇一**進行,不要全做。

| 子場景 | 你會學到 | 適合誰 |
|--------|----------|--------|
| [lit-review](./lit-review/) | 跨多篇論文抽取重點、製作比較矩陣、找 research gap | 正在寫 literature review 或讀 paper 讀到頭痛 |
| [survey-analysis](./survey-analysis/) | 讓 AI 寫 Python 處理 CSV、畫圖、出敘述性統計 | 不會寫 code 但要分析問卷資料 |
| [references](./references/) | 把混亂引用統一格式、找出缺漏與重複 | 引用格式被 supervisor 嫌過 |
| [dissertation](./dissertation/) | 把 outline 變成資料夾結構、給每章 supervisor 視角的回饋 | 論文卡在「不知道下一步寫什麼」 |

## 共通的 Academic Integrity 原則

> AI 是**研究助理**,不是**作者**。

| 可以 ✅ | 不可以 ❌ |
|---------|-----------|
| 整理、抽取、比較資訊 | 直接用 AI 寫的 critical analysis 當作自己的論證 |
| 改格式、找錯、檢查一致性 | 用 AI 編造不存在的 reference |
| 生成腳本處理資料 | 把 AI 的初稿一字不改交出去 |
| 列出可能的論點供你選擇 | 用 AI 假裝理解你其實沒讀過的論文 |

**判斷標準:** 你的 supervisor 看到成品,能不能說「這個 argument 是 [你的名字] 想出來的」?如果不能,那就是 AI 做太多了。
